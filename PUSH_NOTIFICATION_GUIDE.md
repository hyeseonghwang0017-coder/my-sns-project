## 🚀 모바일에서 테스트하기

### 1단계: 앱 빌드
```bash
npm run build
npx cap sync android
```

### 2단계: Android 스튜디오에서 빌드 및 실행
```bash
npx cap open android
```

### 3단계: 로그인 및 테스트
1. 앱을 실행하고 로그인/회원가입
2. 다음 두 가지 중 선택:
   - **옵션 A**: 다른 기기/계정에서 댓글이나 좋아요 달기
   - **옵션 B**: 백엔드에서 테스트 푸시 알림 전송:
     ```bash
     cd backend
     python send_test_push.py "이메일" "제목" "내용"
     ```

### 4단계: 디버그 로그 확인
1. 앱의 주소 표시줄에 접근 가능한 경우: `http://localhost:3000/debug-logs`
2. 또는 모바일에서: 앱 메뉴에서 설정/개발자 옵션으로 이동

## 📱 알림 흐름 (완전한 구조)

```
┌─────────────────┐
│  사용자 A의 앱  │ (로그인됨)
│  Device Token   │ ─────→ 서버 저장
│  저장 완료      │
└─────────────────┘
        ↓
┌─────────────────┐
│  사용자 B가     │
│  A의 게시글에   │
│  댓글 달기      │
└─────────────────┘
        ↓
┌──────────────────────────────┐
│  백엔드                      │
│  1. DB에 알림 저장           │
│  2. A의 device_token 조회   │
│  3. Firebase FCM에 푸시 요청 │
└──────────────────────────────┘
        ↓
┌──────────────────────────────┐
│  Firebase Cloud Messaging    │
│  → A의 기기로 푸시 알림 전송 │
└──────────────────────────────┘
        ↓
┌──────────────────────────────┐
│  사용자 A의 기기             │
│  • 앱이 백그라운드 → 알림 표시 │
│  • 앱이 포그라운드 → onMessage│
└──────────────────────────────┘
```

## ⚠️ 주의사항

1. **Device Token 만료**: 앱을 재설치하거나 앱 데이터를 초기화하면 새 토큰이 발급됨
2. **테스트 시 두 개의 기기/계정 필요**:
   - 기기 A: 게시글 작성자 (알림 받을 사람)
   - 기기 B: 댓글/좋아요 작성자 (알림 보낼 사람)
3. **네트워크 연결**: 앱이 FCM 서버와 통신할 수 있어야 함

## 🎯 다음 단계

알림이 제대로 작동하지 않으면:
1. 디버그 로그를 확인하고 에러 메시지 기록
2. `send_test_push.py`로 직접 푸시 알림 테스트
3. Firebase Console에서 기기 토큰이 올바르게 저장되었는지 확인
4. Android 기기의 설정 > 앱 권한에서 알림 권한 확인
